    Create SQLAlchemy models for the knowledge bank schema:

    1. **backend/app/models/__init__.py**:
       - Import and re-export all models from knowledge.py (Topic, Subtopic, Question, AnswerOption)

    2. **backend/app/models/knowledge.py**:
       - Import Base from app.database
       - All models use UUID primary keys via `Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)`
       - All models have created_at and updated_at timestamp columns

       **class Topic(Base):**
       - __tablename__ = "topics"
       - name: String(100), unique, not null (e.g., "Cardiology", "Pharmacology")
       - discipline: String(50), not null (e.g., "basic_science", "clinical_science", "behavioral_social")
       - description: Text, nullable
       - Relationship: subtopics (one-to-many to Subtopic)
       - Relationship: questions (one-to-many to Question)

       **class Subtopic(Base):**
       - __tablename__ = "subtopics"
       - topic_id: ForeignKey("topics.id"), not null
       - name: String(200), not null (e.g., "Heart Failure", "Beta Blockers")
       - description: Text, nullable
       - Relationship: topic (many-to-one), questions (one-to-many)

       **class Question(Base):**
       - __tablename__ = "questions"
       - topic_id: ForeignKey("topics.id"), not null
       - subtopic_id: ForeignKey("subtopics.id"), nullable
       - stem: Text, not null (the clinical vignette / question text)
       - question_type: String(30), not null, default "single_best_answer" (values: single_best_answer, sequential, drag_and_drop, free_response)
       - usmle_step: String(10), not null (values: "step1", "step2ck", "step3")
       - difficulty: String(10), not null, default "medium" (values: easy, medium, hard)
       - source: String(20), not null, default "curated" (values: curated, ai_generated)
       - explanation: Text, nullable (detailed explanation of the correct answer)
       - learning_objectives: Text, nullable (JSON-encoded list of learning objectives)
       - Relationship: topic (many-to-one), subtopic (many-to-one), answer_options (one-to-many)

       **class AnswerOption(Base):**
       - __tablename__ = "answer_options"
       - question_id: ForeignKey("questions.id", ondelete="CASCADE"), not null
       - label: String(5), not null (e.g., "A", "B", "C", "D", "E")
       - text: Text, not null (the answer option text)
       - is_correct: Boolean, not null, default False
       - explanation: Text, nullable (why this specific option is right or wrong)
       - Relationship: question (many-to-one)

       Add indexes:
       - Index on Question.usmle_step
       - Index on Question.topic_id
       - Index on Question.question_type
       - Index on Question.difficulty

    After creating models, add a script at bottom of main.py (or a separate init_db section) to create tables:
    - In backend/app/main.py, add a startup event that calls `Base.metadata.create_all(bind=engine)` importing Base from database and engine, and importing models to register them.

    Add comments explaining every model, every field, and every relationship.
