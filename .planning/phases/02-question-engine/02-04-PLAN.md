**1. Create `frontend/components/quiz/DragDropView.tsx`:**

"use client" component. JSDoc on every function/interface.

Props:
```typescript
interface DragDropViewProps {
  question: Question;
  onComplete: () => void;
}
```

State:
- `placements: Map<string, string>` — maps draggable item text to target zone
- `isSubmitted: boolean`
- `isCorrect: boolean`

Implementation:
- Parse the stem to extract drag targets. The stem contains "**Target Zones:**" followed by a list. Parse these as the drop zone labels.
- Parse answer_options: each option's `text` contains "item → target" format (e.g., "Metoprolol → Beta-1 selective antagonist"). Split on " → " to get the draggable item name and the correct target zone.
- Render the draggable items on the left and drop target zones on the right
- Use HTML5 Drag and Drop API:
  - Each draggable item: `<div draggable onDragStart={(e) => e.dataTransfer.setData("text", itemText)}>` with `cursor-grab` styling
  - Each target zone: `<div onDragOver={(e) => e.preventDefault()} onDrop={(e) => handleDrop(e, zoneName)}>` with dashed border styling
  - When an item is dropped on a zone, update `placements` Map
  - Visual feedback: items in zones show as small pill/tag elements inside the zone div
  - Items can be dragged out of zones back to the items area (remove from placements)
- Submit button: disabled until all items are placed
- On submit: compare placements to correct mappings from answer_options. Mark each zone green (correct) or red (incorrect)
- After submit: show TeachingPanel for explanation
- Mobile fallback: on small screens, show dropdown selects instead of drag-and-drop (detect `window.innerWidth < 768` or use a `@media` approach with Tailwind `md:hidden` and `hidden md:block`)
