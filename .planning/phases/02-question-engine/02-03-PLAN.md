**1. Create `frontend/components/quiz/SequentialCaseView.tsx`:**

"use client" component. JSDoc on every function and interface.

Props:
```typescript
interface SequentialCaseViewProps {
  questions: Question[];      // All parts of this sequential case, ordered
  onCaseComplete: () => void; // Called when all parts are answered
}
```

State:
- `currentPart: number` — which part the user is on (0-indexed)
- `answers: Map<number, AnswerSubmitResponse>` — submitted answers per part index
- `selectedOption: string | null` — selection for current part
- `isSubmitting: boolean`
- `showWarning: boolean` — whether to show the lock warning modal before proceeding

Behavior:
- Shows all parts vertically, but only the current part is interactive
- Previous parts are displayed in a collapsed/summary view showing: part number, stem (truncated to 100 chars), user's answer, and correct/incorrect indicator (green check or red X)
- Current part shows full stem + AnswerOption components
- Before submitting each part (except the last), show a warning: "You won't be able to change this answer. Are you sure?" with "Confirm" and "Go Back" buttons
  - The warning appears AFTER the user clicks Submit (not before selecting)
  - If user confirms: submit the answer, lock the part, advance to next part
  - If user goes back: dismiss warning, keep selection
- After submitting the LAST part: show TeachingPanel for the last part and call `onCaseComplete()`
- Each locked part has a subtle gray background and `opacity-70`
- Progress indicator: "Part {currentPart + 1} of {questions.length}"
