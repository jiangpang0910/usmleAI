    Create `backend/app/schemas/session.py` with Pydantic models:

    **RecordAnswerRequest**: question_id (UUID), selected_option_label (str), is_correct (bool), topic_id (UUID), difficulty (str), time_spent_seconds (Optional[int])
    **RecordAnswerResponse**: id (UUID), is_correct (bool), created_at (datetime)
    **TopicPerformance**: topic_id (UUID), topic_name (str), total_answered (int), correct_count (int), accuracy (float) — computed as correct_count/total_answered
    **AdaptiveSessionRequest**: question_count (int, default 20, ge=5, le=40), usmle_step (Optional[str])
    **AdaptiveSessionResponse**: session_id (UUID), questions (list — list of question IDs and metadata for frontend to fetch), weak_topics (list[TopicPerformance])
    **PerformanceSummaryResponse**: total_answered (int), overall_accuracy (float), topics (list[TopicPerformance])

    Create `backend/app/routers/sessions.py` with router prefix `/api/sessions` and tag "sessions":

    **POST /api/sessions/record-answer**: Accepts RecordAnswerRequest, creates a UserAnswer row, returns RecordAnswerResponse. This is called after the existing /api/answers/submit so both validation and history happen.

    **GET /api/sessions/performance**: Queries UserAnswer table grouped by topic_id, joins Topic for names, computes accuracy per topic, returns PerformanceSummaryResponse. Use `func.count` and `func.sum(case(...))` for aggregation.

    **POST /api/sessions/adaptive/start**: Accepts AdaptiveSessionRequest. Logic:
    1. Query performance summary (same logic as GET /performance)
    2. Identify weak topics: topics with accuracy < 0.6 (60% threshold per CONTEXT.md)
    3. If no weak topics (new user or all strong), use all topics equally
    4. Create a StudySession with session_type="adaptive"
    5. Query questions prioritizing weak topics: 70% from weak topics, 30% from other topics
    6. Apply the optional usmle_step filter if provided
    7. Randomize within each group using `func.random()` (PostgreSQL)
    8. Limit to question_count total
    9. Return AdaptiveSessionResponse with session_id, question IDs, and weak_topics
