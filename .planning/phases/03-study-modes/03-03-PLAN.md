  <action>
    Create `backend/app/schemas/exam.py` with Pydantic models:

    **EXAM_CONFIGS** — module-level dict constant defining the default exam structure per step:
    ```python
    EXAM_CONFIGS = {
        "step1": {"max_blocks": 7, "questions_per_block": 40, "minutes_per_block": 60},
        "step2ck": {"max_blocks": 8, "questions_per_block": 40, "minutes_per_block": 60},
        "step3": {"max_blocks": 6, "questions_per_block": 40, "minutes_per_block": 60},
    }
    BREAK_POOL_SECONDS = 2700  # 45 minutes total break time across all blocks
    ```
    Comment: Step 3 uses Day 1 MCQ format (6 blocks). Day 2 CCS is not simulated in v1.

    **ExamConfigSchema** (Pydantic): usmle_step (str, must be one of "step1", "step2ck", "step3"), block_count (int, ge=1, le=14, description: how many blocks to run, capped at max_blocks for the step)

    **ExamSessionRequest**: config (ExamConfigSchema)

    **ExamBlockSchema**: block_number (int), question_ids (list[UUID]), time_limit_seconds (int)

    **ExamSessionResponse**: session_id (UUID), blocks (list[ExamBlockSchema]), total_questions (int), break_pool_seconds (int)

    **BlockAnswerItem**: question_id (UUID), selected_option_label (Optional[str] — null means unanswered/timed out), topic_id (UUID), difficulty (str)

    **SubmitBlockRequest**: session_id (UUID), block_number (int), answers (list[BlockAnswerItem]), time_spent_seconds (int)

    **BlockResultItem**: question_id (UUID), selected_label (Optional[str]), correct_label (str), is_correct (bool)

    **SubmitBlockResponse**: block_number (int), total (int), correct (int), results (list[BlockResultItem])

    Create `backend/app/routers/exam.py` with router prefix `/api/sessions/exam` and tag "exam":

    **POST /api/sessions/exam/start**: Accepts ExamSessionRequest. Logic:
    1. Validate usmle_step is in EXAM_CONFIGS
    2. Look up config: questions_per_block, minutes_per_block from EXAM_CONFIGS
    3. Cap block_count at the step's max_blocks
    4. Calculate total questions needed: block_count * questions_per_block
    5. Query Question table filtered by usmle_step, randomized with `func.random()`, limited to total needed
    6. If not enough questions available, use what exists and note in response (partial blocks OK — per CONTEXT.md Claude's discretion on insufficient questions)
    7. Create a StudySession with session_type="exam_simulation", usmle_step=step, config_json=JSON of the config, total_questions=actual count
    8. Split questions into blocks: chunk the question list into groups of questions_per_block
    9. Build ExamBlockSchema for each block with block_number (1-indexed), question IDs, time_limit_seconds = minutes_per_block * 60
    10. Return ExamSessionResponse with session_id, blocks, total_questions, break_pool_seconds=BREAK_POOL_SECONDS

    **POST /api/sessions/exam/submit-block**: Accepts SubmitBlockRequest. Logic:
    1. Load the StudySession by session_id (404 if not found)
    2. For each answer in the request:
       - Look up the question to find the correct option label
       - Determine is_correct: if selected_option_label is None (unanswered), is_correct = False
       - Create a UserAnswer row with session_id, question_id, topic_id, selected_option_label (or "X" for unanswered), is_correct, difficulty
    3. Count correct answers in this block
    4. Update session.correct_count += block correct count
    5. Commit all to DB
    6. Return SubmitBlockResponse with per-question results
